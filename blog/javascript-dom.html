Javascript & the DOM
Javascript and how it works with HTML and CSS

A very basic way to describe the relationship between these 3 programming languages is as if they are a football team.

HTML would be the players. You need the players on the pitch to be able to do anything. The team could not really exist without them.

CSS would be what the pitch and the players look like. What colours are the kit? Are there any fancy designs mowed into the grass?

Javascript would be the manager/coach. Someone who is able to change the instructions for a specific player or the team as whole. Take one player off and put another on.

HTML on its own would work as a webpage but by using CSS and Javascript, we are able to make it look a lot nicer as well as be a lot more functional.

The DOM

The Document Object Model (or DOM) is a hierarchical system that allows for the organisation and access of aspects within a webpage. It is often thought of as a family tree which is especially apt when referring to parent and child nodes within it.

The DOM provides an Application Programming Interface (API) that a scripting language (in this case Javascript) uses to interact with the document. In that sense, the DOM kind of acts like a translator for the Javascript to communicate with the rest of the document.

An obvious example of it is when we are calling an element to do something with, as shown below:

let pageHeading = document.getElementById('pageHeading')

Here we are defining a variable within Javascript. The "document" part is instructing the code to take the information from somewhere within the Document node.

Arrays and Objects

An array is a range of values given in the form of an ordered list.
An object is made up of key-value pairs, where the key tells us the aspect and the value gives us the detail.

An array would be like a team of football players

let team = ['thomas', 'shane', 'david', 'jamie', 'brad'];

And you would access it using the index number of the player that you want

console.log(team[2]);

Would return "david" in the console

An object would be the details for each individual program

let goalkeeper = { name: 'thomas'; shirtNumber: 1; height: '192cm'; dominantFoot: 'left' }

We would acces this using the name of the object and the key

console.log(goalkeeper.dominantFoot);

Would return "left" in the console

Control Flow

Control flow is the order of operations that the code runs through. Javascript runs top to bottom by default. This can be altered by things such as conditions, loops or functions by telling the code to call an aspect form somewhere else in the script to be run at that time.

Functions

Functions are a set of actions that are placed within a group and given a name. This is useful if you need to use the same set of actions multiple times across a script. It saves on time as you do not need to type it out each time. It also means that if you ever need to change the way the set of actions is working, you only need to change it on one part of the script instead of combing through the whole file to change each set that was typed out.

Loops

Loops are something that Javascript uses to repeat a process until a condition is met.
We can use cleaning rooms in your house as an example.

let messyRooms = ['kitchen', 'lounge', 'masterBedroom', 'guestBedroom', 'bathroom'];

let cleanRooms = [];

for( i=0; i < messyRooms.length; i++){
cleanRooms.push(messyRooms[i]);
console.log(cleanRooms[i] + ' is now clean')
};

This loop would start by 'cleaning' the first room in the array, then cycle through the “messyRooms” array until all of them have been cleaned. It would tell us in the console after each room has been cleaned.
